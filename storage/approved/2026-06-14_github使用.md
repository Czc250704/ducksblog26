第一步：注册 GitHub 账号（已有请登录（Sing in）后跳转下一步）

打开 github.com，要是没账号就点中间那个 Sign up，注册方式有Google，Apple，email。个人推荐使用email
Email就输入你的电子邮箱地址，路xxx@xxx.com
Password输入你要的密码，建议8个字符以上，一定要有数字和小写字母，不然不行。
Username输入用户名。注意：用户名采用英文最好，可以省去麻烦。
Your Country/Region选择China就行

注意，这用户名后面有大用，它会变成你网站域名的一部分，所以最好别起一串乱码，比如叫duck就比duck1863强。
接下来就是通过验证码的环节，验证码如果看不懂就开启浏览器翻译，看中文提示。如果不会的话，过个十几分钟再试，Github不同时间段的验证码方式是不一样的。

---

第二步：创建一个叫“特殊名字”的仓库

登录进去之后，看页面右上角，你的头像左边会有一个 “+” 号，点它，然后选 New repository。

这时候会跳到一个新建仓库的页面，好多英文别怕，只看几个地方：

1. Repository name 下面那个输入框，这步最关键，弄错你就白干了。
      填的名字必须是：你的用户名.github.io
      比如你用户名是 laowang，那就填 laowang.github.io。
      注意：全小写，中间没空格。如果填对了，输入框下面会冒出来一行绿字儿，提示“你正在创建用户站点”，看到这个就稳了。
2. 下面那个 Description (描述) 爱写不写，随便。
3. 最重要的是，下面单选框一定选 Public——公开。 因为 GitHub Pages 在免费账号下，只有公开仓库才能用，你选 Private 的话后面就找不到按钮。（除非你给钱升级，但咱不花那冤枉钱）
4. 底下有个 “Add a README file” 的勾选框，可以先不勾，后头我们自己加东西。
      然后，直接点绿油油的 Create repository 按钮。

好了，现在空仓库建好了，页面会跳进去。

---

第三步：往仓库里扔一个网页文件

现在你在这个空仓库的页面里，应该能看到一堆文件列表是空的，中间有个区域写着 “Quick setup”。别管它，咱们自己动手。

找到那个蓝色的 Add file 按钮（可能在右上角附近），点一下，选 Create new file。

接下来，给文件起名字：

· 在上方一个窄窄的输入框里，把默认的 “Name your file…” 删掉，写上：index.html
    这个名字不能改，它代表你网站的首页。
· 下面大大的编辑框就是写网页内容的地方。咱先随便整点最简单的，你直接复制粘贴下面这行：
  ```html
  <h1>嘿！我居然搞出来一个网站！</h1>
  ```
  要是你想洋气点，写 Hello World 也行。
· 页面往下拉，你会看到 Commit new file 这区域。上面两个输入框不用填，那是给改代码的大神写备注的，咱不鸟它。
    直接点那个绿色的 Commit new file 按钮。

好，这就算把网页文件存进去了，页面会自动回到文件列表，你能看到 index.html 躺在那儿。

---

第四步：打开 GitHub Pages 开关

现在离成功就差最后一口劲儿了。

在这个仓库页面顶部，找到一排标签：Code、Issues、Pull requests…… 往右看，有一个 Settings（扳手图标）。点它。

进 Settings 之后，左边竖排菜单很长，往下使劲划拉，找到 Pages 这个选项（它在 “Code and automation” 这个大组下面，别点岔了）。

现在右边会出现 “GitHub Pages” 的设置。咱们看 Build and deployment 那一块。

· 在 Source 下面，有个下拉框，默认可能显示 “Deploy from a branch”，如果不是，你把它选成这个。
· 然后下边会多出来 Branch 的选项，它旁边有个按钮，很可能显示 “None”。点它，选 main。
· 再右边有个文件夹选择，默认应该是 / (root)，就保持这个别动。
· 完事儿点那个 Save 按钮。

点完 Save，页面可能会自己刷新一下，然后你就能在 Pages 设置区域的最上方，看到一行字，大概长这样：

Your site is live at https://你的用户名.github.io

如果没马上看到，别慌，喝口水，等个一两分钟再刷新。GitHub 那头还得喘口气部署呢。最慢也就三五分钟。

---

第五步：打开你的网站！

看到那个 https://xxxx.github.io 的链接之后，直接点它，或者复制到浏览器里打开。
只要不出意外，你刚才写的 “嘿！我居然搞出来一个网站！” 就会神奇地出现在网页正中央。恭喜，你有了一个全世界都能访问的自己的地盘！

---

要是没成功，常见几个坑，自己对号入座：

1. 仓库名写错：比如用户名是 LaoWang，但仓库名写成了 laowang.githu.io 或者多了个字母。回去 Settings 里最上面可以改仓库名（Rename），改正确了 Pages 会自动重新部署。
2. 仓库是 Private（私有的）：去 Settings 页面最底下 “Danger Zone” 那里，点 “Change visibility”，改成 Public，然后再去 Pages 设置里重新保存一下。
3. 分支没选对：确认 Branch 选了 main，而不是其他什么 gh-pages。
4. 没等够：刚点完 Save 就打网址，多半是 404，等个两分钟刷新就好了。
5. 文件没起对名字：必须是 index.html，不是 index.html.txt 也不是 Index.html。

---

之后你要是想改网页内容，就点进那个 index.html 文件，然后点右上角的铅笔图标编辑，改完再点那个绿色的 Commit 按钮。等大概一两分钟刷新你的网站，就能看到新内容了。想加图片、改颜色什么的，自己慢慢折腾。
