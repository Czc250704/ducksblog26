# Duck's Blog · Public Studio  
## 用户故事与行为路径 v1.0

**项目名称**：ducksblog26H2  
**域名**：duckpublic.qd.je

## 一、角色定义

| 角色 | 账号 | 核心目标 |
|------|------|----------|
| 访客 | 无需登录 | 浏览内容、听音乐、发表评论 |
| 二级管理员 | admin1-4 | 创建分类、上传内容（需审核） |
| 最高管理员 | duck | 管理一切、审批内容、删除分类/管理员 |

---

## 二、用户故事

### 2.1 访客

**故事 1：浏览博客内容**
> 作为一个访客，我想打开网站就能看到所有已发布的内容，不需要登录，这样我可以快速获取信息。

**故事 2：查看最新动态**
> 作为一个访客，我想在右侧边栏看到最近有哪些新内容上传和审核通过，这样我不会错过更新。

**故事 3：评论动态**
> 作为一个访客，我想对某条动态发表评论（跟帖），和其他访客交流看法，不需要注册账号。

**故事 4：听音乐**
> 作为一个访客，我想在浏览博客时播放背景音乐，音乐来自博主上传的歌曲列表。

**故事 5：预览文档**
> 作为一个访客，我想点击文件后直接在网页上查看内容（MD文档、PPT、Word），不需要下载到本地。

**故事 6：按分类筛选**
> 作为一个访客，我想按分类查看内容，只看到我感兴趣的主题。

---

### 2.2 二级管理员（admin1-4）

**故事 7：登录后台**
> 作为一个二级管理员，我需要用账号密码登录后台，登录后我才能看到上传入口和管理面板。

**故事 8：创建分类**
> 作为一个二级管理员，我想创建新的内容分类，比如“技术分享”或“生活随笔”，这样内容更有条理。

**故事 9：上传文件**
> 作为一个二级管理员，我想上传 MD、PPT、Word 或 TXT 文件，上传后文件进入待审核状态，等待最高管理员批准。

**故事 10：查看自己的上传记录**
> 作为一个二级管理员，我想看到我上传了哪些文件，以及哪些已通过审核、哪些还在等待。

---

### 2.3 最高管理员（duck）

**故事 11：登录超级管理面板**
> 作为最高管理员，我需要用专属账号登录，登录后我要看到完整的控制台，包括审批待处理文件、管理分类、管理其他管理员。

**故事 12：审批文件**
> 作为最高管理员，我打开审批界面能看到所有待审核的文件列表，我可以选择“通过”或“拒绝”。通过后文件自动发布到前台，并同步到 GitHub 仓库。

**故事 13：删除分类**
> 作为最高管理员，如果某个分类不再需要，我可以直接删除它，分类下的所有内容也会被移除。

**故事 14：删除其他管理员**
> 作为最高管理员，如果某个二级管理员不再需要权限，我可以将其账号删除。

**故事 15：解除调试限制**
> 作为最高管理员，我登录后应该自动解除 F12 和右键菜单的禁用，方便我调试和检查问题。

**故事 16：查看所有动态**
> 作为最高管理员，我想在后台看到完整的操作日志：谁上传了什么、谁通过了审核、谁发表了什么评论。

## 四、核心路径总结

### 典型用户旅程

| 用户 | 入口 | 核心操作 | 终点 |
|------|------|----------|------|
| 访客 | 直接访问 | 浏览 → 预览文档 → 评论 | 离开网站 |
| 二级管理员 | 登录 | 创建分类 → 上传文件 → 等待审核 | 登出 |
| 最高管理员 | 登录 | 审批文件 → 管理分类/管理员 | 登出 |

### 文件完整生命周期

```
二级管理员上传
       │
       ▼
   pending 目录 + status='pending'
       │
       ▼
最高管理员在审批界面查看
       │
       ├──────────────────┐
       ▼                  ▼
    通过                 拒绝
       │                  │
       ▼                  ▼
移动到 approved     删除文件
更新 status         更新 status='rejected'
更新 manifest
Git commit & push
       │
       ▼
前台显示 + 生成动态
```
